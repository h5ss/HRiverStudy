### 1단계 : 시스템 상황 분석
- 메인서비스만 정리
#### 1. 상품관리
- 상품등록 
  - 쇼핑몰에 상품을 등록하기위해 셀픽에 상품을 등록하는 메뉴
  - 상품명, 브랜드 (브랜드코드 : 셀픽에 브랜드명과 코드를 등록 미리 해야하지만 스킵하자), 원산지 (브랜드코드랑 동일), 제조사, 상품판매가, 과세구분, 미성년자구매가능여부, 판매상태, 상품설명, 이미지
- 상품조회/수정
***
- 셀픽 고객 --- 셀픽 
- 셀픽에 상품등록 하는 기능이다
- 쇼핑몰 하위 목록중 하나로 보임
#### 2. 쇼핑몰관리 (유스케이스 시스템으로 보아야할듯?)
- 연동설정 : 셀픽과 쇼핑몰 연동
- 주문수집설정 : 쇼핑몰 주문들 수집, 확정처리
- 주문확인/처리 : 확정한 수집한 주문들 출고처리
- 상품등록 : 쇼핑몰에 등록할 상품을 셀픽에 등록
- 쇼핑몰 상품 등록 : 등록 상품과 쇼핑몰을 선택 후 , 쇼핑몰에 정보 전달
- 쇼핑몰 상품조회/수정 : 쇼핑몰에 등록된 상품 조회 수정
***
- 셀픽 --- 쇼핑몰 연동 
#### 3. 주문관리
- 쇼핑몰관리 하위 항목
- 주문수집설정 : 쇼핑몰주문을 수집함 (결제일자기준)
- 쇼핑몰 주문 확정 버튼 : 셀픽 주문 상태 변경(발주 확정) -- 고객이 판매자가 주문확인한것을 알수 있음

#### 4. 발주관리
- 발주등록: 발주일자, 매입거래처, 창고, 발주구분 
- 발주 확정 취소 기능 존재
- 발주확정: 발주등록 건 확정 하는 것 --> 입고관리에서 발주 확정된거 확인가능
- 발주조회 : 발주등록 조회
- 
#### 5. 배송관리
- 송장자동출력 설정 : 택배사 선택, 발송지정보(창고 정보 : 보내는 사람, 번호, 주소)
- 송장 출력(주문 확인 창) : 주문확정 된 곳에 송장 자동 생성 버튼이 있음
- 
#### 6. 매장관리
- 매출관리 
  - 매출등록 : 소매매장에서 판매된 단품을 등록합니다. 매출등록시 매장의 재고가 차감되고,
    매출이 발생합니다.
    
  - 매출조회 : 매장별 매출을 조회 할 수 있습니다.
    
  - 매출현황 : 매장별 매출현황을 확인 할 수 있습니다.
    
  - 결제수단별조회 : 매출등록시 입력한 결제수단별로 조회 할 수 있습니다.
    
  - 시간대별조회 : 매출등록한 시간대별로 매출현황을 확인 할 수 있습니다.
#### 7. 마스터 통합관리
- 하나의 마스터 계정으로 다수의 셀픽계정 관리
#### 8. 통합재고관리
- 재고현황 
  - 창고와 매장의 재고를 한번에 확인
  - 
#### 9. 영업분석관리
- 매출계획대비실적
  - 공급단가와 매출단가로 조회하여 계획관리 등록한 내용과 비교하여 달성률 출력
#### 10. 클레임/CS관리
- Q&A/클레임 수집
  - 쇼핑몰에서 발생한 클레임(취소/교환/반품) 및 Q&A를 수집
- Q&A 처리
  - 수집된 Q&A에 답변을 저장 한 후 쇼핑몰로 전송
- CS이력현황
  -  수집된 Q&A, 클레임의 이력을 확인

### 5단계 : 유스케이스 명세서 작성

1. -  유스케이스명 : 발주 확정
   - 액터명 : 입출고 담당자
   - 유스케이스 개요 : 입출고 담당자는 제품의 발주를 확정한다.
   - 사전 조건 : 현황관리 담당자가 주문 확정을 한 제품이다.
   - 이벤트 흐름
     - 정상 흐름
     1. 제품을 사전에 발주등록을 한다.
     2. 현황관리 담당자가 주문 확정을 했는지 확인한다.
     3. 주문 확정을 했다면 등록된 발주 정보를 통해 발주를 확정한다.
     - 선택 흐름
     1. 발주등록의 정보를 확인하기 위하여 발주 조회를 한다.
     2. 발주 확정에 오류가 있다면 발주 확정 취소를 한다.
     

2. - 유스케이스명 : 송장 출력
   - 액터명 : 입출고 담당자
   - 유스케이스 개요 : 입출고 담당자가 주문확정 된 제품의 송장을 출력을 한다. (배송을 보내기 위한 작업)
   - 사전 조건 : 송장자동출력설정을 통해 택배사 및 발송지 정보를 저장해야 한다.
   - 이벤트 흐름
     - 정상 흐름
     1. 송장자동출력설정을 통해 택배사 및 발송지 정보를 저장한다.
     2. 주문 확정을 기다린다.
     3. 주문 확정이 된 제품의 송장 출력을 진행한다.


3. - 유스케이스명 : 주문 확정
   - 액터명 : 현황관리 담당자
   - 유스케이스 개요 : 현황관리 담당자가 주문 상태를 확인 후 확정을 한다.
   - 사전 조건 : 주문수집설정을 통하여 쇼핑몰에서 주문 정보들을 수집한다.
   - 이벤트 흐름
     - 정상 흐름
     1. 주문수집설정을 통해 쇼핑몰의 주문을 수집한다.
     2. 주문 정보를 확인 후, 주문 확정을 한다.
     3. 주문 확정을 입출고 담당자에게 알린다.
     

4. - 유스케이스명 : 상품 등록
   - 액터명 : 현황관리 담당자
   - 유스케이스 개요 : 현황관리 담당자가 상품 정보를 등록한다.
   - 사전 조건 : 필수 작성 정보를 준비해야한다.
   - 이벤트 흐름
     - 정상 흐름
     1. 판매하고자 하는 상품 정보를 준비한다.
     2. 상품 등록에 작성한다.
     - 선택 흐름
     1. 등록된 상품의 정보를 조회 한다.
     2. 등록된 상품의 정보 오류가 있다면 상품 수정을 한다.
     

5. - 유스케이스명 : 재고 현황
   - 액터명 : 현황관리 담당자
   - 유스케이스 개요 : 모든 곳의 재고를 확인 한다.
   - 사전 조건 : 상품이 등록되어 있어야 한다.
   - 이벤트 흐름
     - 정상 흐름
     1. 상품 등록을 한다.
     2. 재고현황을 통하여 재고 확인을 한다.

6. - 유스케이스명 : 매출현황조회
   - 액터명 : 현황관리 담당자
   - 유스케이스 개요 : 매출의 현황을 조회를 하여 확인한다.
   - 사전 조건 : 상품등록이 되어 있어야 한다.
   - 이벤트 흐름
     - 정상 흐름
     1. 상품 등록을 한다.
     2. 매출 등록을 하여 재고 수량과, 매출을 발생시킨다.
     3. 매출현항조회를 통하여 현재 매출 상황을 확인한다.
     - 선택 흐름
     1. 시간대별조회를 통하여 시간대에 따른 매출현황을 확인한다.
     2. 결제수단별조회를 통하여 결제수단별로 매출현황을 확인한다.

7. - 유스케이스명 : Q&A 처리
   - 액터명 : 클레임 담당자
   - 유스케이스 개요 : 쇼핑몰에서 발생한 클레임을 처리한다.
   - 사전 조건 : 클레임 정보가 수집되어 있어야 한다.
   - 이벤트 흐름
     - 정상 흐름
     1. 클레임 정보를 쇼핑몰에서 수집해온다.
     2. 수집한 클레임 정보를 답변을 작성 후 저장한다.
     3. 쇼핑몰로 정보를 전송한다.
     - 선택 흐름
     1. 수집한 클레임 이력들을 확인한다.
